Refusal in Audio MLLMs is Mediated by a Single Direction

We show that the 1-dimensional refusal subspace identified in text-input LLMs transfers to the audio modality in Qwen2-Audio-7B — enabling cross-modal mechanistic interventions that bypass refusal on audio inputs.

Key Findings

Activation Space

PCA Visualisations

Residual stream activations at last-token position, projected to 2D via PCA. Hover over points to see the instruction. Click audio-modality points to play the spoken instruction. Arrows show the refusal direction vectors in PCA space.

Linear Separability

Linear Probe Results

A logistic regression probe trained on raw residual stream activations with 5-fold cross-validation. Accuracy far above chance indicates linear separability of harmful, harmless, and jailbreak categories in the model's internal representations.

Layer Modality Classes CV Accuracy Chance Δ Chance
Key insight: Audio 3-class probe achieves 98.9% accuracy at layer 31, with jailbreak category F1 = 1.00 — the most linearly separable class. Jailbreak instructions that elicited compliance cluster geometrically distinct from both harmful and harmless inputs, while failed jailbreaks co-locate with harmful instructions.
Causal Intervention

Cross-Modal Ablation

We ablate the text-derived refusal direction from audio activations at all layers during generation. If the same 1D subspace mediates refusal across modalities, this intervention should bypass refusal on audio inputs — without any audio-specific fine-tuning.

Methodology

How It Works

🎙️
TTS
Instructions → 16kHz waveforms via Kokoro/gTTS
🔊
Whisper
Whisper-Large-V2 encoder → 1280-dim states
🔗
Projector
Qwen2-Audio trained projector → 4096-dim LLM space
🧠
HookedTransformer
Qwen1.5-7B shell + Qwen2-Audio LM weights via TransformerLens
📐
Difference of Means
Refusal direction = mean(harmful) − mean(harmless) at last token
✂️
Ablation
Project out refusal direction from all residual stream positions

The Qwen2-Audio backbone (hidden_size=4096, 32 layers, 32 heads) is architecturally identical to Qwen1.5-7B-Chat, enabling direct weight injection into TransformerLens. Audio tokens are injected via hook_embed, allowing all downstream hooks to operate unchanged on the combined sequence.

Non-Technical Guide

How to Update This Page

All results are stored in the CONFIG object at the top of this file. No coding knowledge is needed — just open the file in any text editor and change values.

To Add a New Dataset

  1. 1 Open index.html in any text editor (TextEdit, VS Code, Notepad)
  2. 2 Find the datasets: section in CONFIG (search for "datasets:")
  3. 3 Copy one block between { } and change the values — name, n (number of samples), type, status ("complete"/"partial"/"pending"), note
  4. 4 Save and refresh in browser. The card appears automatically.

To Add New Probe Results

  1. 1 Find probe_results: in CONFIG
  2. 2 Add a new line: { layer: 14, modality: "Audio", classes: 4, acc: 0.95, std: 0.02, chance: 0.25 },
  3. 3 Save — the new row appears in the table automatically.

To Add an Interactive Plot

  1. 1 Place the .html plot file in the same folder as index.html
  2. 2 Find plots: in CONFIG and add: { title: "My Plot", subtitle: "description", path: "myplot.html", available: true }
  3. 3 Save — the plot iframe appears automatically.
